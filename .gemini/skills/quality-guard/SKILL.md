# Quality Guard Workflow

이 스킬은 프로젝트의 코드 품질을 유지하고, 배포 시 발생할 수 있는 잠재적 에러를 사전에 방지하기 위해 사용됩니다.

## 1. 검증 항목 및 명령어
작업 완료 보고 전, 다음 단계들을 순차적으로 실행하십시오.

1.  **타입 체크 (Type Check)**
    *   명령어: `pnpm exec tsc --noEmit`
    *   목적: TypeScript 타입 오류가 없는지 확인합니다.
2.  **린트 체크 (Lint Check)**
    *   명령어: `pnpm lint`
    *   목적: 코드 스타일 및 ESLint 규칙 준수 여부를 확인합니다.
3.  **빌드 체크 (Build Check)**
    *   명령어: `pnpm build`
    *   목적: Next.js 정적 빌드가 성공적으로 완료되는지 확인합니다. (Github Pages 배포 가능 여부 확인)

## 2. 실행 지침
*   모든 명령어는 프로젝트 루트 디렉토리에서 실행하십시오.
*   각 단계에서 에러가 발생하면, 즉시 중단하고 에러를 수정한 뒤 다시 검증을 시작하십시오.
*   **빌드 체크**는 시간이 다소 소요될 수 있으므로, 대규모 작업 완료 후에 실행하는 것을 권장합니다.

## 3. 결과 보고 형식
검증이 완료되면 다음과 같은 형식으로 사용자에게 보고하십시오.

> ### ✅ 최종 품질 검증 결과
> *   **타입 체크:** 통과 / 실패 (실패 시 에러 내용 요약)
> *   **린트 체크:** 통과 / 실패 (실패 시 에러 내용 요약)
> *   **빌드 체크:** 통과 / 실패 (실패 시 에러 내용 요약)
> 
> [AI의견] 모든 검증이 완료되었습니다. 이제 안전하게 작업을 마무리할 수 있습니다.
