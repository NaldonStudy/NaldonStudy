# Pre-commit Validator Workflow

이 스킬은 코드를 원격 저장소에 올리기 전, 프로젝트의 품질과 컨벤션을 최종적으로 점검하기 위해 사용됩니다.

## 1. 통합 검증 항목
이 스킬은 다음 항목들을 순차적으로 검사합니다.

1.  **기술적 결함 검사**
    *   **타입 체크**: `pnpm exec tsc --noEmit`
    *   **린트 체크**: `pnpm lint`
    *   **빌드 체크**: `pnpm build` (정적 배포 가능 여부 확인)
2.  **구조 및 컨벤션 검사**
    *   **Atomic Design**: `components/` 폴더 구조 준수 여부
    *   **폴더 명명 규칙**: `kebab-case` 준수 여부
    *   **함수/컴포넌트 명명 규칙**: 파일명과 컴포넌트 이름 일치 여부

## 2. 사용 방법

```bash
node .gemini/skills/pre-commit-validator/scripts/validate-all.cjs
```

## 3. 실행 지침
*   모든 단계가 **통과(Success)**되어야 안전하게 커밋/푸시를 진행할 수 있습니다.
*   실패한 항목이 있다면 해당 에러 메시지를 확인하여 수정한 후 다시 실행하십시오.
*   `pnpm build` 과정은 시간이 다소 소요될 수 있으므로 인내심을 가지고 기다려 주십시오.

## 4. 결과 보고 형식
